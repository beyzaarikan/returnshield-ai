# ReturnShield Data Dictionary

This document describes the main data tables, important fields, and data provenance used in the ReturnShield data/ML and agent pipeline.

Core provenance labels used in this project:

| Data status | Meaning |
|---|---|
| Open/raw data | Field directly coming from an open dataset |
| Derived | Field computed from open/raw data inside the notebooks |
| Simulated | Field generated in a controlled way for the MVP/demo |
| Agent output | Risk, explanation, or action output generated by Notebook 06 |
| Diagnostic/config | Supporting artifact used to explain model or notebook behavior |

> Note: `data/raw/` and `data/processed/` are local pipeline folders and are not committed to GitHub. The repository tracks notebook code and selected artifacts under `outputs/`, such as plots, metrics, diagnostics, summaries, and agent outputs.

---

## 1. Data sources

| Source | Purpose | Data status | Note |
|---|---|---|---|
| UCI Online Retail | Transaction layer, return/cancel proxy label, baseline risk ranking | Open/raw data | Negative quantities and cancellation-like records are used to derive a return/cancel proxy label. |
| ModCloth Clothing Fit | Fit, size, length, quality, rating, and review-based signals | Open/raw data | Used to build fashion-specific fit and review signal layers. |
| RentTheRunway Clothing Fit | Fit, size, length, quality, rating, and review-based signals | Open/raw data | Used together with ModCloth to strengthen the fashion signal layer. |
| Mock cart generation | Cart-level demo input generation | Simulated | UCI transactions and fashion signals are not treated as true row-level product matches. The mapping is controlled for demo use only. |

## 8. Risk Agent final output fields

Main demo output:

```text
outputs/06_returnshield_agent_demo/returnshield_agent_cart_scores.csv
```

| Field | Description | Data status | Usage |
|---|---|---|---|
| `risk_score` | Final Risk Agent score | Agent output | Main dashboard/backend demo risk score. Range: 0-1. |
| `risk_percentile_rank` | Percentile rank within the demo cart batch | Agent output | Used to derive `risk_level`. |
| `risk_level` | Risk level: `low`, `medium`, or `high` | Agent output | Frontend risk badge and dashboard filter. |
| `label_backed_risk_score` | Transaction-model anchor score | Agent output | Shows the label-backed component of the risk score. |
| `contextual_evidence_score` | Contextual score from strong fit/review/cart evidence | Agent output | Shows multi-signal contextual contribution. |
| `context_signal_count` | Number of fit/review/cart context evidence flags | Agent output | Diagnostic and explanation support. |
| `risk_input_coverage_weight` | Available input weight used in risk scoring | Agent output | Shows that missing layers are not treated as low risk. |
| `top_reasons` | List of reason codes | Agent output | Frontend/backend explanation field. Stored as a JSON string. |
| `reason_details` | Reason code, severity, and message details | Agent output | Dashboard explanation panel. Stored as a JSON string. |
| `suggested_action` | Selected action code | Agent output | Action Agent output. |
| `dashboard_message` | Explanation/action message for the dashboard or customer-facing UI | Agent output | Message card in the demo UI. |

### Possible `suggested_action` values

| Value | Meaning |
|---|---|
| `show_size_guidance_before_checkout` | Show size/fit guidance before checkout |
| `highlight_color_and_product_images` | Highlight product images and color details |
| `show_review_and_product_detail_summary` | Show review summary and product details |
| `confirm_duplicate_variant` | Ask the user to confirm a repeated variant |
| `show_return_policy_and_checkout_confirmation` | Show return policy or checkout confirmation |
| `standard_checkout` | Continue with the standard checkout flow |

### Possible `top_reasons` values

| Reason code | Meaning |
|---|---|
| `high_transaction_model_rank` | Transaction model rank is high |
| `fit_layer_signal` | General fit layer signal is elevated |
| `fit_or_size_issue_signal` | Fit or size mismatch signal is elevated |
| `length_issue_signal` | Length issue signal is elevated |
| `quality_issue_signal` | Quality/material issue signal is elevated |

| `rare_color_issue_review_flag` | Color/visual mismatch appears as an upper-tail review flag |
| `low_rating_signal_when_available` | Low-rating issue signal is elevated when rating exists |
| `two_sizes_same_product` | The same product appears with two different sizes in the cart |
| `duplicate_variant` | The same variant appears more than once |
| `large_cart_size` | Cart size is high within the demo batch |
| `dominant_transaction_model_signal` | Transaction model is the strongest available layer |
| `dominant_fit_signal` | Fit is the strongest available layer |
| `dominant_review_text_signal` | Review text is the strongest available layer |
| `dominant_cart_behavior_signal` | Cart behavior is the strongest available layer |

## 10. Important limitations

- The ReturnShield MVP is not a production model trained on one real e-commerce company's end-to-end order, review, and return dataset.
- UCI Online Retail and ModCloth/RentTheRunway are not merged as true row-level product matches.
- Mock cart and mapping fields are controlled demo simulation artifacts.
- `risk_score` should be interpreted as an MVP risk-ranking and agentic decision-support score, not as a calibrated probability of return.
- Fit, review, and cart signals support the risk score and mainly provide value through `top_reasons`, `suggested_action`, and `dashboard_message`.
- Agent/LLM-style outputs are not treated as automatic ground truth. They are controlled decision-support outputs.
